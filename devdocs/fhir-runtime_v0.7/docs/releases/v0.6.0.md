# fhir-runtime v0.6.0

Release date: 2026-03-09

## Highlights

- IG Package & Canonical Resolution (`STAGE-4`) released
- New `NpmPackageLoader` — load conformance resources from extracted FHIR IG NPM packages
- `PackageManager` — register, discover, and manage multiple IG packages
- Cross-package canonical URL resolution with version awareness (`url|version`)
- Dependency graph resolution with topological sorting and circular dependency detection
- `.index.json` fast lookup with filesystem scan fallback
- US Core IG (v9.0.0) successfully loaded and validated (70 SDs, 20 ValueSets, 4 CodeSystems)
- Backward compatible — all 3,128 v0.5.0 tests still pass

## Added

### Package Module (STAGE-4)

- New `src/package/` module added to the public package surface
- Added `NpmPackageLoader` class implementing `StructureDefinitionLoader`:
  - `load()` / `canLoad()` — standard loader interface for CompositeLoader integration
  - `getManifest()` / `getIndex()` — package metadata access
  - `getEntries()` / `getEntriesByType()` — resource enumeration
  - `loadAllStructureDefinitions()` / `loadAllValueSets()` / `loadAllCodeSystems()` — bulk loading
  - `loadResource()` — load any resource by canonical URL
  - `resolveCanonical()` — resolve URL to file entry
  - `.index.json` support with filesystem scan fallback

### PackageManager

- Added `PackageManager` class for multi-package management:
  - `registerPackage()` — register an extracted IG package
  - `discoverPackages()` — auto-discover packages in a cache directory
  - `resolveDependencies()` — build dependency graph with topological ordering
  - `resolveCanonical()` — cross-package canonical URL resolution
  - `resolveAllByType()` — find all resources of a type across packages
  - `createLoader()` — create CompositeLoader for FhirContext integration

### Manifest & Index Parsers

- Added `parsePackageManifest()` / `parsePackageManifestFromString()` — parse `package.json`
- Added `parsePackageIndex()` / `parsePackageIndexFromString()` — parse `.index.json`
- Added `filterIndexByResourceType()` — filter index entries

### Dependency Resolver

- Added `buildDependencyGraph()` — recursive dependency resolution
- Added `topologicalSort()` — dependency-order sorting
- Added `findMissingDependencies()` — detect unresolved dependencies
- Added `CircularPackageDependencyError` — thrown on circular dependencies

### Canonical Resolver

- Added `parseCanonicalUrl()` — split `url|version` format
- Added `resolveCanonical()` — version-aware resolution across loaders
- Added `resolveAllByType()` — enumerate all resources of a type

### Type Definitions

- Added `PackageManifest`, `PackageIndex`, `PackageIndexEntry`
- Added `NpmPackageLoaderOptions`, `PackageManagerOptions`
- Added `PackageInfo`, `DependencyGraph`, `DependencyNode`, `CanonicalResolution`
- Added `CONFORMANCE_RESOURCE_TYPES` constant

## Compatibility

- All v0.5.0 public API remains unchanged
- `NpmPackageLoader` implements `StructureDefinitionLoader` — works with `CompositeLoader` and `FhirContextImpl`
- Zero runtime dependencies (unchanged)
- Node.js >=18.0.0 (unchanged)
- ESM primary + CJS compatibility (unchanged)

## Upgrade Notes

1. Install as usual — no breaking changes
2. Use `NpmPackageLoader` to load IG packages:
   ```typescript
   const loader = new NpmPackageLoader('/path/to/package');
   const sd = await loader.load(canonicalUrl);
   ```
3. Use `PackageManager` for multi-package workflows:
   ```typescript
   const manager = new PackageManager();
   await manager.registerPackage('/path/to/us-core/package');
   const compositeLoader = manager.createLoader();
   ```

## Known Limits

- No .tgz download support (extraction must be done externally)
- No FHIR Package Registry client (remote queries belong to CLI/server layer)
- Version ranges in dependencies not supported (exact match only)
- Browser compatibility requires abstracting `node:fs` calls
